# VideoNotes Pro — Promotion Playbook

## Best + Easiest Growth Plan (Low Social, Low Waste)

A practical, step-by-step system to launch and grow VideoNotes Pro without being heavily active on social media.

---

## Who This Is For

- You like AI + tech and building products
- You are **not** active on social media
- You want growth tactics that are durable, low-maintenance, and measurable
- You want better conversions using psychology, but without spammy or manipulative tactics

---

## North Star Strategy (Keep It Simple)

Your growth engine should be:

1. **Search-driven acquisition** (Chrome Web Store + Google + YouTube)
2. **Conversion-optimized listing** (clear value + clear paid upgrade reason)
3. **In-product monetization moments** (upgrade asks at the right time)
4. **Light automation** (AI-assisted content and posting workflows)
5. **Review flywheel** (ratings create trust, trust creates installs)

If you execute this consistently for 8–12 weeks, you’ll build momentum with minimal busywork.

---

## Ground Rules (So You Don’t Waste Time)

- Avoid trying to be on every platform
- Avoid daily posting pressure
- Avoid fake scarcity or manipulative tactics
- Focus on channels where users already search for solutions
- Run weekly experiments and keep what works

**Time budget target:** 3–5 hours/week

# Phase 1 — Positioning + Offer (Day 1)

## Step 1: Lock your one-line value proposition
Use this as the core line across store listing, landing page, and content.

**Primary line:**
> Take timestamped notes on YouTube and jump back to exact moments instantly.

**Secondary line:**
> Built for students, researchers, and creators who never want to lose the important part of a video.

## Step 2: Define your 3 buyer segments
Keep copy specific to these:

1. **Students** — lecture notes and exam review
2. **Researchers** — source tracking and citation prep
3. **Creators/Professionals** — extracting ideas from long-form videos

## Step 3: Clarify free vs paid psychology
Your paid plan should feel like a natural next step, not a surprise.

- **Free:** enough to prove value (3 videos)
- **Paid:** removes friction and unlocks serious use (unlimited + export/import)

This uses **commitment/consistency psychology**: once users get value from free, upgrading is a logical continuation.

# Phase 2 — Chrome Web Store Optimization (Day 1–2)

This is your most important acquisition channel.

## Step 1: Optimize listing title and short description
- Keep keyword-rich language users actually type

## Step 2: Build screenshot sequence as a story
Use 3–5 screenshots max. Each screenshot should answer one question:

1. What is this?
2. How fast is it?
3. Why is it better than doing it manually?
4. What does Pro unlock?

## Step 3: Write conversion-focused long description
Structure:
1. Problem sentence
2. 3 core benefits

5. Privacy reassurance

## Step 4: Add trust markers

- “No account required to start”

This reduces perceived risk and increases install conversion.

# Phase 6 — Psychology for Better Paid Conversion (Ethical)

Use these principles responsibly.

## 1) Clarity over cleverness
People pay when the value is obvious in seconds.
- State exact outcome: “Find any important moment instantly”

## 2) Loss aversion (ethical framing)
Show what users lose without Pro:
- “Without export/import, your workflow stays trapped in-browser”

## 3) Progress principle
Users are more likely to pay after early wins.
- Encourage first 2–3 notes quickly during onboarding

## 4) Anchoring
Show annual plan value compared to monthly.
- “$24.99/year (~$2.08/mo) vs $3.99 monthly”

## 5) Social proof
Display real user quotes and review snippets (with permission).
- “Saved me hours for exam prep”

## 6) Risk reduction
Reduce anxiety before purchase:
- clear refund policy (if applicable)
- transparent privacy statements

## 7) Timing of upgrade ask
Prompt when pain is highest and value is proven:
- at free 3-video limit
- when clicking export/import

**Avoid:** fake countdowns, fake stock limits, misleading claims.

Yes, this should be in your system explicitly.

Include two CTAs:

## Step 2: Mid-cycle value reminder (Day 7)
Trigger a lightweight in-product message after active usage:

"Quick win: you’ve already captured X notes this month. Keep building your searchable study library."

This uses progress reinforcement and reminds them why they paid.

## Step 3: Renewal reminder + annual upgrade anchor (Day 21–26)
Show a polite reminder to monthly users:

"Thanks again for being Pro. Your plan renews soon so you keep unlimited notes + exports.  
Want the best value? Switch to Annual and save more over time."

- "Switch to Annual"

- If renewal succeeds: "Thanks for continuing your support — Pro remains active."

## Step 5: Guardrails (important)
- Do not spam reminders every session
- Add "Don’t show again this month"
- Keep reminders value-focused, not fear-focused

# Phase 10 — Thank You + Share Link + Coupon/Referral System

Yes, this is possible. The easiest path is to start simple, then automate.

## Option A (Fast MVP): Manual referral code flow
1. Generate a unique referral code per Pro user (example: `WES-7H2K`)
2. Create a share URL with query params:
  - `https://YOUR_SITE/?ref=WES-7H2K`
3. On your landing page, detect `ref` and display coupon instructions
4. Buyer applies a promo code at checkout
5. You credit the referrer manually at first (discount, bonus month, or perk)

This is quickest and works even before full backend automation.

## Option B (Automated): Stripe + webhook referral tracking
1. Keep unique `ref` in landing-page URL
2. Persist click + referral code in a lightweight database
3. Pass referral metadata into checkout (where supported)
4. Use Stripe webhook events (`checkout.session.completed`, renewals, cancellations)
5. Auto-award reward when paid conversion is confirmed

## Suggested reward models
- **Give/Get % discount**: Referrer gets 20%, friend gets 20%
- **Give/Get fixed**: Referrer gets $5 credit, friend gets $5 off
- **Bonus month**: Referrer gets +1 month Pro after successful referral

## Best first implementation for your setup
Start with **Option A** for launch week. Upgrade to Option B after you have 50+ paid users.

## "Thank you, please share" copy
"Thanks for supporting VideoNotes Pro ❤️  
Want to help another student or creator? Share your link and they’ll get a discount."

# Phase 11 — Separate MacBook Air M1 as Local AI Workflow Machine

Yes — a separate MacBook Air M1 is a good setup for local automation.

## What this gives you
- Runs content-generation workflows without using your main machine
- Lets you test local LLM prompts privately
- Can schedule recurring jobs (weekly post drafts, changelog-to-content, etc.)

## Practical stack
- `Ollama` for local model serving
- `n8n` for automations/workflows
- Optional: lightweight dashboard or scripts for approvals

## Step-by-step setup
1. Install Homebrew (if not installed)
2. Install Ollama and pull one efficient model (for M1):
  - `llama3.1:8b` or `qwen2.5:7b`
3. Install `n8n` (Docker or npm mode)
4. Create workflow:
  - Trigger: weekly schedule
  - Input: changelog + one metric + one user quote
  - LLM step (Ollama): generate 5 post variants + 1 release note
  - Output: write to Google Sheet/Notion/Markdown for approval
5. Add human review step before publishing
6. (Optional) Connect Buffer for scheduled posting

## Can it run OpenClaw?
If your OpenClaw workflow is CLI/Python-based, yes, the M1 can run it as a worker machine.  
Use the same pattern: scheduled trigger → local generation → review queue → optional publish.

## M1 performance guidance
- Prefer 7B/8B models for reliability and speed
- Keep context windows modest
- Run overnight batch jobs for heavy generation

Create 5 variants of a Chrome Web Store short description (max 132 chars), each optimized for a different audience: students, researchers, creators, professionals, and general productivity users. Keep language clear, no hype, include one benefit and one action phrase.

## 2) Generate screenshot headline text
Prompt:
> Write 20 short screenshot headlines (4–8 words) for a Chrome extension that captures YouTube timestamps and notes. Tone: practical, fast, trustworthy. Avoid clickbait.

Prompt:

## 6) Monthly renewal reminder copy (ethical)
Prompt:
> Write 10 short in-product reminder messages for monthly subscribers of VideoNotes Pro. Tone: appreciative, helpful, non-pushy. Include one annual-plan value anchor in 4 of the variants. Keep each under 140 characters.

Prompt:
> Create 12 "thank you + share" messages for Pro subscribers. Goal: encourage sharing a referral link without sounding spammy. Include one-line versions and two-line versions. Mention a friend discount in half the variants.

## 8) Referral landing page copy
Prompt:
> Write landing page copy for users arriving from a referral link for VideoNotes Pro. Include: headline, short proof statement, friend discount callout, CTA button text, and a one-line trust/privacy reassurance.
