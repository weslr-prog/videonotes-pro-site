# PART B — V1.1: SMART IMPORT & LINKBACK

The signature V1.1 feature: users can import previously exported notes and click timestamps to be taken back to the original video at the exact moment.

---

## Feature Overview

| What | Details |
|------|---------|
| Problem | Exported notes are static text. Users can't get back to the video from them. |
| Solution | 1. Upgrade the export format to include clickable YouTube URLs with timestamps. 2. Add an Import button that reads a previously exported file and loads the notes back into the extension. 3. Each imported note's timestamp is a clickable link that opens the original video at the correct time. |
| Who benefits | Students reviewing study notes weeks later, researchers revisiting source videos, anyone who exported notes and wants to jump back. |
| Pro only? | Yes — both import and the enhanced export are Pro features. |

---

## Phase 7: Upgrade Export Format — Add Video URLs + Timestamp Links (45 minutes)

### What
Currently the export format looks like:
```markdown
## Video Title
- **02:34** — My note text here
```

Upgrade it to include the YouTube URL with a `t=` timestamp parameter so each note links back to the exact moment:
```markdown
## Video Title
🔗 https://www.youtube.com/watch?v=VIDEO_ID

- [**02:34**](https://www.youtube.com/watch?v=VIDEO_ID&t=154) — My note text here
```

### File to edit
`sidepanel.js` — the `exportNotes()` function

### Copilot Prompt
> "In my exportNotes() function in sidepanel.js, upgrade the Markdown output format. For each video section:
> 1. After the `## Video Title` heading, add a line that says `🔗 VIDEO_URL` using the url field from the first note in that video's array (strip any existing `t=` parameter from it)
> 2. Change each note bullet from `- **02:34** — note text` to `- [**02:34**](VIDEO_URL&t=SECONDS) — note text` where SECONDS is the note's timestampSeconds value (floored to integer) and VIDEO_URL is the clean base URL of the video
> 3. If a note has no url field, fall back to the plain format without a link"

### Verification
- [ ] Export a file and open it in a Markdown viewer (VS Code preview, GitHub, etc.)
- [ ] Each video section has a `🔗` link line under the heading
- [ ] Each timestamp is a clickable Markdown link
- [ ] Clicking a timestamp link in the Markdown opens the YouTube video at the correct time
- [ ] Notes without URL data still export cleanly (no broken links)

## Phase 8: Build the Import File Parser (1 hour)

### What
Write a function that reads an exported `.md` file and parses it back into the same note object structure used by `chrome.storage.local`.

### File to edit
`sidepanel.js` — new function `parseImportedMarkdown(markdownText)`

### Expected input format
```markdown
## Video Title
🔗 https://www.youtube.com/watch?v=VIDEO_ID

- [**02:34**](https://www.youtube.com/watch?v=VIDEO_ID&t=154) — My note text here
- [**05:12**](https://www.youtube.com/watch?v=VIDEO_ID&t=312) — Another note
```

### Expected output structure
```javascript
{
  "notes_video_VIDEO_ID": [
    {
      id: Date.now(),
      videoTitle: "Video Title",
      videoId: "VIDEO_ID",
      timestampSeconds: 154,
      timestampFormatted: "02:34",
      noteText: "My note text here",
      url: "https://www.youtube.com/watch?v=VIDEO_ID"
    },
    // ... more notes
  ]
}
```

### Copilot Prompt
> "Write a function called parseImportedMarkdown(markdownText) that takes a string of Markdown content (exported from VideoNotes Pro) and returns an object where each key is a storage key like 'notes_video_VIDEOID' and each value is an array of note objects. The Markdown format has:
> - `## Video Title` headings for each video section
> - An optional `🔗 URL` line with the video URL
> - Bullet points in the format `- [**MM:SS**](URL&t=SECONDS) — note text` or the legacy format `- **MM:SS** — note text`
> Parse the video ID from the URL using URLSearchParams. Convert MM:SS back to total seconds. Generate a unique id using Date.now() + index. Handle both the new linked format and the old plain format gracefully."

### Edge cases to handle
- Legacy export format (no links, no 🔗 line) — should still import correctly
- Missing video ID — use a fallback key based on the video title
- Empty sections — skip them
- Malformed lines — skip them without crashing

### Verification
- [ ] Parses the new linked export format correctly
- [ ] Parses the old plain export format correctly
- [ ] Returns correct storage key structure
- [ ] Handles edge cases without errors

## Phase 10: Wire Up Import Logic (1 hour)

### What
Connect the file picker to the parser, save imported notes to storage, and show the user what was imported.

### File to edit
`sidepanel.js` — new function `importNotes()` + event listeners

### Copilot Prompt
> "Write an importNotes() function for my Chrome extension side panel. It should:
> 1. First call checkProStatus(). If the user is not Pro, call openUpgradePage() and return.
> 2. Programmatically click the hidden file input with id 'import-file-input' to open the file picker.
> 3. Listen for the file input's 'change' event. When a file is selected:
>    a. Read the file contents using FileReader.readAsText()
>    b. Pass the text to parseImportedMarkdown() to get the parsed notes object
>    c. For each storage key in the parsed result, merge the notes with any existing notes under that key in chrome.storage.local (avoid duplicates by checking timestampSeconds + noteText combinations)
>    d. Show an alert telling the user how many notes were imported across how many videos
>    e. Call loadNotes() to refresh the display
>    f. Reset the file input value so the same file can be re-imported if needed
> 4. Add the event listener for the import button (id 'import-btn') in the DOMContentLoaded handler."

### Duplicate detection logic
When merging imported notes with existing notes, skip an imported note if an existing note has the same `timestampSeconds` AND the same `noteText` (trimmed). This prevents double-imports.

### Verification
- [ ] Clicking Import opens file picker (Pro user only)
- [ ] Non-Pro user gets upgrade prompt instead
- [ ] Selecting a valid `.md` file imports notes correctly
- [ ] Imported notes appear in the notes list for the current video (if applicable)
- [ ] Duplicate notes are not created on re-import

### Copilot Prompt
> "Modify my renderNotes() function and jumpToTimestamp() function in sidepanel.js:

## Suggested Approach

### Option A: Ship V1.0 first, then build V1.1 (Recommended)
1. Complete Phases 1–6 (go-live) — get V1.0 published and earning revenue
2. While waiting for Chrome Web Store review (1–5 days), start building Phases 7–13 locally
3. Once V1.0 is approved and live, finish and submit V1.1 as an update

### Option B: Build everything, ship once
1. Complete all 14 phases
2. Submit V1.1 as the first published version
3. Downside: delays your launch and revenue by the time it takes to build import/LinkBack

**Option A is recommended** because:
- You start earning sooner
- V1.0 is already fully tested and working
- Real user feedback might influence how you build V1.1
- Google's review queue means shipping sooner = live sooner
